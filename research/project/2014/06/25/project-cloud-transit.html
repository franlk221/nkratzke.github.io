Research context

Actual cloud computing related research focuses often Service Science related problems and therefore concentrates on aspects how to design, operate and deliver agile cloud based services according to a broad (economical as well as technological) understanding of the service term.

Beside common security, governance and cost forecast concerns especially technological vendor lock-in worries often come along with cloud computing which is especially true for small and medium-sized enterprises (Kra2011a, Kra2011b).

But astonishing little research activities focus a technological point of view how to overcome technical vendor lock-in problems in cloud computing.

Therefore this technological point of view is addressed particularly by this research project.

Research objective

Actual state of the art cloud based software services are often characterized by a highly implicit technological dependency on underlying hosting cloud infrastructures. The project Cloud TRANSIT investigates how to design cloud based software services to reduce technological dependencies on underlying cloud infrastructures.

By designing and developing a generic cloud service description language it should be possible to define secure, transferable and elastic services of typical complexity being deployable to any IaaS cloud infrastructure. It furthermore should be possible to migrate these services from one private or public cloud infrastructure to another (not necessary compatible) cloud infrastructure.

Especially transferability reduces cloud infrastructure related technical vendor lock-in problems for small and medium sized enterprises substantially.

The project will therefore

  • analyse several common public and private cloud infrastructures
  • derive requirements for a generic cloud service description language
  • for cloud services of typical complexity (distributed, load balanced, auto scaling)
  • implement a prototypic implementation of the description language
  • to provide a proof-of-concept.
