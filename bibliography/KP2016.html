Publication [KP2016]

ClouNS - A Reference Model for Cloud-Native Applications

Abstract: There are still prevalent vendor lock-in apprehensions after 10 years of cloud computing. Standardization would be an answer. But standardization processes are slow. Cloud computing standardization coverage decreased from 70% in 2006 down to 20% in 2016. This is mainly due to cloud services being faster released by public cloud providers than they can be standardized. As a result, cloud-native applications are vulnerable for vendor lock-in. We inspected motivations for cloud-native applications (and related architecture and infrastructure approaches), service release notes by public cloud service providers, industrial cloud standards, open-source bottom up and formalized deployment approaches to overcome vendor lock-in. All results indicate that cloud standards concentrate on a very small but basic subset of popular cloud services often called IaaS. All other service categories seem to foster vendor lock-in situations. This might sound disillusioning at first. But the presented ClouNS reference model for cloud-native applications shows that it is possible to rely only on this small subset of well standardized IaaS services to overcome vendor lock-in. It can be used for codifying cloud technologies as well as guiding technology identification, classification, adoption, research and development processes for cloud service provider portable cloud-native applications.
